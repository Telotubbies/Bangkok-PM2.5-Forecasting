# 🌍 อธิบาย STC-HGAT Model - การพยากรณ์ PM2.5 กรุงเทพฯ

## 📊 ภาพรวม Model

Model ของเราชื่อ **STC-HGAT** (Spatio-Temporal Contrastive Heterogeneous Graph Attention Networks) ซึ่งเป็น AI model ที่ออกแบบมาเพื่อพยากรณ์ค่า PM2.5 โดยคำนึงถึง:
- **พื้นที่** (Spatial) - สถานีต่างๆ ในกรุงเทพฯ
- **เวลา** (Temporal) - การเปลี่ยนแปลงตามเวลา
- **ความสัมพันธ์** (Graph) - ความเชื่อมโยงระหว่างสถานี

---

## 🎯 ตัวอย่างการทำงานจริง

### **สถานการณ์:** พยากรณ์ PM2.5 ที่สถานี Din Daeng

**ข้อมูลที่ Model ใช้:**

1. **ข้อมูลจากสถานี Din Daeng เอง:**
   - PM2.5 ปัจจุบัน: 45 µg/m³
   - PM10: 80 µg/m³
   - อุณหภูมิ: 32°C
   - ความชื้น: 65%
   - ความเร็วลม: 5 km/h
   - ทิศทางลม: 90° (ลมพัดจากทิศตะวันออก)

2. **ข้อมูลจากสถานีใกล้เคียง:**
   - **Bang Khen** (ทางเหนือ, 8 km):
     - PM2.5: 52 µg/m³
     - ลมพัดจากทิศตะวันออกเฉียงเหนือ
   - **Bangkok** (ทางใต้, 5 km):
     - PM2.5: 38 µg/m³
     - ลมพัดจากทิศตะวันออก
   - **Lat Yao** (ทางตะวันออก, 6 km):
     - PM2.5: 58 µg/m³ ⚠️ สูง!
     - ลมพัดจากทิศตะวันออก

## 🧠 Model คิดยังไง?

### **Step 1: Spatial Analysis (วิเคราะห์พื้นที่)**

Model จะดูว่า **ลมพัดมาจากทิศไหน** และ **สถานีไหนอยู่ทางทิศนั้น**:

```
ทิศทางลม: 90° (ตะวันออก)
↓
ลมพัดจาก Lat Yao (ทางตะวันออก) → Din Daeng
↓
Lat Yao มี PM2.5 = 58 µg/m³ (สูง!)
↓
⚠️ มลพิษจะพัดมาที่ Din Daeng!
```

**HyperGAT (Spatial Module) ทำอะไร:**
- คำนวณ **attention weight** ให้สถานีที่อยู่ทางทวนลม (upwind)
- Lat Yao ได้ weight สูง เพราะ:
  - ✅ อยู่ทางตะวันออก (ตรงกับทิศทางลม)
  - ✅ ระยะทางใกล้ (6 km)
  - ✅ PM2.5 สูง (58 µg/m³)
- Bang Khen และ Bangkok ได้ weight ต่ำกว่า เพราะไม่ได้อยู่ทางทวนลม

**สูตรการคำนวณ:**
```
Attention Weight = f(ระยะทาง, มุมลม, ค่า PM2.5)

สำหรับ Lat Yao:
- ระยะทาง: 6 km → weight = 1/6² = 0.028
- มุมลม: 0° (ตรงกับทิศลม) → weight = 1.0
- PM2.5: 58 → weight สูง
→ Total weight สูง!
```

### **Step 2: Temporal Analysis (วิเคราะห์เวลา)**

Model จะดูว่า **PM2.5 เปลี่ยนแปลงยังไงในอดีต**:

```
เวลา:        08:00  09:00  10:00  11:00  12:00 (ตอนนี้)
PM2.5:        35     38     42     45     ?
แนวโน้ม:      ↗️     ↗️     ↗️     ↗️     ⚠️ กำลังเพิ่ม!
```

**HGAT (Temporal Module) ทำอะไร:**
- วิเคราะห์ **pattern ตามเวลา**:
  - 🌅 เช้า (06:00-09:00): PM2.5 เพิ่มขึ้น (รถติด)
  - ☀️ กลางวัน (10:00-14:00): PM2.5 สูงสุด
  - 🌆 เย็น (15:00-18:00): PM2.5 เริ่มลด
  - 🌙 กลางคืน (19:00-05:00): PM2.5 ต่ำสุด

- วิเคราะห์ **วันในสัปดาห์**:
  - จันทร์-ศุกร์: PM2.5 สูง (รถติด)
  - เสาร์-อาทิตย์: PM2.5 ต่ำกว่า

- วิเคราะห์ **ฤดูกาล**:
  - มีนาคม-เมษายน: PM2.5 สูง (ไฟป่า)
  - พฤษภาคม-ตุลาคม: PM2.5 ปานกลาง (ฝน)

### **Step 3: Multi-Scale Temporal (วิเคราะห์หลายระดับเวลา)**

Model จะดู pattern ในช่วงเวลาต่างๆ **พร้อมกัน**:

```
ระดับ 1 (1 ชั่วโมง):
  11:00 → 12:00: เพิ่ม 3 µg/m³
  แนวโน้มระยะสั้น: ↗️ เพิ่มขึ้น

ระดับ 2 (3 ชั่วโมง):
  09:00 → 12:00: เพิ่ม 7 µg/m³
  แนวโน้มระยะกลาง: ↗️ เพิ่มขึ้นต่อเนื่อง

ระดับ 3 (6 ชั่วโมง):
  06:00 → 12:00: เพิ่ม 10 µg/m³
  แนวโน้มระยะยาว: ↗️ ช่วงเช้าถึงเที่ยง
```

### **Step 4: Cross-Attention Fusion (รวมข้อมูล)**

Model จะ**รวมข้อมูล spatial และ temporal**:

```
Spatial Info:
  "ลมพัดจาก Lat Yao (PM2.5=58) มาที่ Din Daeng"

Temporal Info:
  "ช่วงเช้าถึงเที่ยง PM2.5 กำลังเพิ่มขึ้น"

Cross-Attention:
  ✅ ลมพัดมลพิษจาก Lat Yao
  ✅ เวลานี้เป็นช่วงที่ PM2.5 เพิ่ม
  ✅ อากาศนิ่ง (ลมอ่อน 5 km/h)
  → ⚠️ PM2.5 จะเพิ่มขึ้นมาก!
```

### **Step 5: Prediction (พยากรณ์)**

Model จะ**พยากรณ์ค่า PM2.5 ในอนาคต**:

```
ปัจจุบัน (12:00): 45 µg/m³

พยากรณ์:
  +1 ชั่วโมง (13:00): 52 µg/m³ ↗️
  +3 ชั่วโมง (15:00): 48 µg/m³ ↘️ (เริ่มลด)
  +1 วัน (พรุ่งนี้ 12:00): 42 µg/m³

เหตุผล:
  - ช่วง 12:00-14:00: PM2.5 สูงสุดของวัน
  - หลัง 14:00: เริ่มลดลง
  - พรุ่งนี้: ลมแรงขึ้น → มลพิษกระจาย
```

## 🌬️ การคำนวณผลกระทบของลม

### **ตัวอย่างจริง:**

**สถานการณ์:** Din Daeng, ลมพัดจากทิศตะวันออก 90° ความเร็ว 5 km/h

**สถานีรอบข้าง:**

| สถานี | ทิศทาง | ระยะทาง | PM2.5 | มุมลม | Weight |
|-------|--------|---------|-------|-------|--------|
| **Lat Yao** | ตะวันออก (90°) | 6 km | 58 | 0° ✅ | **สูง** |
| Bang Khen | เหนือ (0°) | 8 km | 52 | 90° | ต่ำ |
| Bangkok | ใต้ (180°) | 5 km | 38 | 90° | ต่ำ |

**การคำนวณ Weight:**

```python
# สำหรับ Lat Yao (ทางตะวันออก)
bearing = 90°  # ทิศทางจาก Din Daeng ไป Lat Yao
wind_dir = 90°  # ลมพัดจากทิศตะวันออก

angle_diff = |bearing - wind_dir| = |90 - 90| = 0°
wind_weight = cos(0°) = 1.0  # ✅ ตรงกับทิศลม!

distance = 6 km
distance_weight = 1 / 6² = 0.028

pm25 = 58
pm25_weight = 58 / 100 = 0.58

total_weight = 1.0 × 0.028 × 0.58 = 0.016  # สูง!
```

```python
# สำหรับ Bang Khen (ทางเหนือ)
bearing = 0°  # ทิศทางจาก Din Daeng ไป Bang Khen
wind_dir = 90°  # ลมพัดจากทิศตะวันออก

angle_diff = |0 - 90| = 90°
wind_weight = cos(90°) = 0.0  # ❌ ไม่ตรงกับทิศลม!

total_weight ≈ 0  # ต่ำมาก!
```

**ผลลัพธ์:**
- Lat Yao มีอิทธิพลมาก → PM2.5 จาก Lat Yao จะพัดมาที่ Din Daeng
- Bang Khen มีอิทธิพลน้อย → PM2.5 จาก Bang Khen ไม่พัดมา

## 📊 Features ที่ Model ใช้ (18 Features)

### **1. PM2.5 Data (6 features):**
```
- pm2_5_ugm3: ค่า PM2.5 (µg/m³)
- pm10_ugm3: ค่า PM10
- no2_ugm3: ไนโตรเจนไดออกไซด์
- o3_ugm3: โอโซน
- so2_ugm3: ซัลเฟอร์ไดออกไซด์
- co_ugm3: คาร์บอนมอนอกไซด์
```

### **2. Weather Data (6 features):**
```
- temperature_2m: อุณหภูมิ (°C)
- relative_humidity_2m: ความชื้น (%)
- precipitation: ปริมาณฝน (mm)
- wind_speed_10m: ความเร็วลม (km/h) ⭐
- wind_direction_10m: ทิศทางลม (°) ⭐
- surface_pressure: ความกดอากาศ (hPa)
```

### **3. Temporal Features (6 features):**
```
- hour_sin, hour_cos: ชั่วโมงในวัน (0-23)
- dow_sin, dow_cos: วันในสัปดาห์ (0-6)
- doy_sin, doy_cos: วันในปี (1-365)
```

**Model แบบ Time Series ธรรมดา:**
```
Input: PM2.5 ที่สถานี Din Daeng ย้อนหลัง 30 ชั่วโมง
↓
LSTM/Transformer
↓
Output: พยากรณ์ PM2.5 ที่ Din Daeng

❌ ปัญหา: ไม่รู้ว่าสถานีอื่นมีค่าเท่าไร
❌ ปัญหา: ไม่รู้ว่าลมพัดจากทิศไหน
```

**Model ของเรา (Graph Neural Network):**
```
Input: 
  - PM2.5 ที่ทุกสถานี (79 สถานี)

  - ความสัมพันธ์ระหว่างสถานี
↓
HyperGAT (Spatial) + HGAT (Temporal)
↓
Output: พยากรณ์ PM2.5 ที่ทุกสถานี

✅ รู้ว่าสถานีไหนมีค่าสูง
✅ รู้ว่าลมพัดมาจากทิศไหน
✅ คำนวณว่ามลพิษจะพัดไปไหน
```
